AWS DMS

Create EC2 Security Group for replication instance

  1. Truy cập vào dịch vụ EC2
  2. Ở thanh điều hướng, chọn Security Groups, nhấn nút Create security group.
    1. Ở mục Basic details
      1. Nhập Security group name
      2. Nhập Description
      3. Chọn VPC đã khởi tạo
    2. Ở mục Inbound rules, nhất nút Add rule và thêm 1 rule như sau:
      1. Type: Custom TCP hoặc MySQL/Aurora (nếu sử dụng port mặc định)
      2. Port range: port mà MySQL server ở on-premise đang sử dụng
      3. Source: IP address mà MySQL server ở on-premise đang sử dụng
    3. Tiến hành khởi tạo

Sau khi khởi tạo thành công, ở Security Group của Aurora MySQL chúng ta sẽ thêm 1 inbound rule tương tự như trên với Source chính là security group đã khởi tạo này để mà Aurora MySQL cho phép truy cập từ replication instance.

Create replication instance

  1. Truy cập vào dịch vụ DMS.
  2. Ở thanh điều hướng, chọn Replication instances, nhấn nút Create replication instance:
    1. Ở mục Settings
      1. Nhập Name
      2. Nhập Description
    2. Ở mục Instance configuration
      1. Chọn Instance class (có thể chọn .medium hoặc .large trở lên)
      2. Engine version: ít nhất là 3.4.x trở lên
      3. Multi AZ: Production workload
    3. Ở mục Storage: ít nhất 50GB để chứa cache
    4. Ở mục Connectivity and security
      1. VPC: chọn VPC đã khởi tạo
      2. Replication subnet group: chọn default-vpc-
      3. Chọn Public accessible nếu không có đường kết nối private tới on-premise. Đối với use-case hiện tại đã có thiết lập AWS VPN S2S nên replication instance sẽ kết nối xuống on-premise thông qua VPN.
      4. (Advanced Settings) VPC security groups: chọn group đã khởi tạo.
      5. (Advanced Settings) AWS KMS key: aws/dms
    5. Tiến hành khởi tạo

Prepare replication endpoints

Sau khi đã khởi tạo thành công replication instance, chúng ta sẽ tạo 2 endpoint lần lượt với kiểu Source endpoint (trỏ về MySQL server ở on-premise) và Target endpoint (trỏ về Aurora MySQL).

  1. Truy cập vào dịch vụ DMS.
  2. Ở thanh điều hướng, chọn Endpoints, nhấn nút Create endpoint:
    1. Ở mục Endpoint type, chọn loại endpoint
    2. Ở mục Endpoint configuration
      1. Nhập Endpoint identifier
        1. Đối với trường hợp Source: giả sử tên của MySQL server
        2. Đối với trường hợp Target: giả sử tên của Aurora MySQL cluster
      2. Chọn Source engine
        1. Đối với trường hợp Source: MySQL
        2. Đối với trường hợp Target: Amazon Aurora MySQL
      3. Nhập Access to endpoint database
        1. Đối với trường hợp Source: chọn Provide access information manually và nhập lần lượt các thông tin cần thiết về truy cập.
        2. Đối với trường hợp Target: chọn AWS Secrets Manager và nhập ARN của Secret.
    3. Ở mục Test endpoint connection (optional): tiến hành test kết nối từ replication instance đã khởi tạo đến thông tin endpoint đã điền.
    4. Nếu status là Successful, tiến hành khởi tạo. Nếu không, chúng ta cần kiểm tra kết nối AWS VPN S2S lẫn thiết bị ở phía on-premise nếu có đang thiết lập các network policies.

Prepare before moving to migration task

Trước khi khởi tạo migration task, chúng ta cần chuẩn bị 1 S3 bucket và 1 IAM role.

  • Với S3 bucket, đây sẽ là nơi chứa các Assessment Reports.
  • Với IAM role, đây sẽ là thành phần cấp quyền truy cập S3 bucket cho dịch vụ DMS.

S3 bucket

  1. Truy cập vào dịch vụ S3.
  2. Ở thanh điều hướng, chọn Buckets, nhấn nút Create bucket.
    1. Ở mục General configuration:
      1. Nhập Bucket name
      2. Chọn AWS Region tương ứng
    2. Ở mục Block Public Access settings, chọn Block all public access:
    3. Ở mục Default encryption
      1. Encryption key type: Amazon S3-managed keys (SSE-S3)
      2. Bucket Key: Enable
    4. Tiến hành khởi tạo

IAM Role

  1. Truy cập vào dịch vụ IAM.
  2. Ở thanh điều hướng, chọn Roles, nhấn nút Create role.
    1. Step 1:
      1. Ở mục Trusted entity type, chọn AWS service
      2. Ở mục Use case, chọn DMS
    2. Step 2, ở mục Add permissions, chọn permission policy chính là AmazonS3FullAccess (tuy nhiên, để tuân thủ best practice thì chúng ta nên khởi tạo 1 custom IAM policy với permisions vừa đủ để truy cập)
    3. Step 3, ở mục Role details, nhập Role name và tiến hành khởi tạo.

Đây là custom IAM policy để thay thế AWS managed policy ở trên.

{
   "Version":"2012-10-17",
   "Statement":[
      {
         "Effect":"Allow",
         "Action":[
            "s3:PutObject",
            "s3:DeleteObject",
            "s3:GetObject",
            "s3:PutObjectTagging"
         ],
         "Resource":[
            "arn:aws:s3:::<s3-bucket-name>/*"
         ]
      },
      {
         "Effect":"Allow",
         "Action":[
            "s3:ListBucket",
            "s3:GetBucketLocation"
         ],
         "Resource":[
            "arn:aws:s3:::<s3-bucket-name>"
         ]
      }
   ]

Create migration task

  1. Truy cập vào dịch vụ DMS.
  2. Ở thanh điều hướng, chọn Database migration tasks, nhấn nút Create task:
    1. Ở mục Task configuration
      1. Nhập Task identifier
      2. Chọn Replication instance đã khởi tạo
      3. Chọn Source database endpoint đã khởi tạo
      4. Chọn Target database endpoint đã khởi tạo
      5. Migration type: Migrate existing data and replicate ongoing changes
    2. Ở mục Task settings
      1. Target table preparation mode: Drop tables on target
      2. Stop task after full load completes: Don't stop
      3. Include LOB columns in replication: Full LOB mode
      4. Task logs: chọn Turn on CloudWatch logs
    3. Ở mục Table mappings: đối với mục này, chúng ta sẽ thêm các selection rules nhằm chọn source và table hoặc loại đi một số table không cần thiết cho quá trình migration. Ngoài ra, còn có các transformation rules để thực hiện việc chuyển đổi dữ liệu.
    4. Ở mục Premigration assessment, chọn Enable premigration assessment run.
      1. Nhập Premigration assessment run name
      2. Assessments to run: chọn tất cả
      3. Assessment report storage: chọn S3 bucket đã khởi tạo
      4. IAM role: chọn 1 IAM role đã khởi tạo
    5. Tiến hành khởi tạo

Sau khi khởi tạo thành công, chúng ta tiến hành quá trình migration.

Đầu tiên, chúng ta sẽ thực hiện bước đánh giá thông qua Premigration asessments, nhấn nút Actions và chọn Create premigration assessment.

Nếu quá trình đánh giá thuận lợi, chúng ta sẽ tiến hành bắt đầu bằng việc nhấn nút Actions và chọn Restart/Resume.
