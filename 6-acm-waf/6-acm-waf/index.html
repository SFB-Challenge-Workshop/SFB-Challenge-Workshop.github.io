AWS WAF & ACM

AWS ACM

Chúng ta sẽ cần ít nhất 2 certificate ở 2 vùng khác nhau

  • Để áp dụng cho các Load Balancer, chọn region là ap-southeast-1 (nơi đang triển khai ứng dụng)
  • Để áp dụng cho các CloudFront distribution, chọn region us-east-1

Request SSL certificates

Sau đây là các bước chúng ta request 1 certificate miễn phí ở region đang triển khai ứng dụng.

  1. Truy cập vào dịch vụ ACM
  2. Ở thanh điều hướng chọn Request certificate:
    1. Ở mục Certificate type, chọn Request a public certificate
    2. Ở mục Domain names, nhập các domain names tương ứng
    3. Ở mục Validation method, tuỳ vào chính sách, chúng ta có thể chọn DNS validation - recommended hoặc Email validation.
    4. Ở mục Key algorithm, chọn RSA 2048
    5. Tiến hành khởi tạo

Import SSL certificate

Sau đây là các bước chúng ta sử dụng certificate đã được issue ở nơi khác, chẳng hạn như từ Comodo hay Google.

  1. Truy cập vào dịch vụ ACM
  2. Ở thanh điều hướng chọn Import certificate:
    1. Ở mục Certificate details, chúng ta cần nhập chính xác các thông số
      1. Certificate body
      2. Certificate private key
      3. Certificate chain
    2. Nếu các thông tin điền là chính xác, dịch vụ sẽ hiển thị thông tin liên quan đến Expiration
    3. Tiến hành khởi tạo

AWS WAF

Chúng ta sẽ cần ít nhất 2 web acls ở 2 vùng khác nhau

  • Để áp dụng cho các Load Balancer, chọn region là ap-southeast-1 (nơi đang triển khai ứng dụng)
  • Để áp dụng cho các CloudFront distribution, chọn region Global

Chúng ta sẽ lặp lại các bước sau để khởi tạo 2 web acls như đã miêu tả ở trên.

  1. Truy cập vào dịch vụ WAF
  2. Ở thanh điều hướng chọn Web ACLs, chọn Region tương ứng và nhấn nút Create web ACL
    1. Step 1:
      1. Ở mục Web ACL details
        1. Nhập Name
        2. Nhập CloudWatch metric name
        3. Resource type
          1. Đối với CloudFront: chọn CloudFront distributions
          2. Đối với Load Balancer: chọn Regional resources (Application Load Balancer, API Gateway, AWS AppSync, Amazon Cognito User Pools)
      2. Ở mục Associated AWS resources - optional, chúng ta sẽ bỏ qua tạm thời do chưa khởi tạo các thành phần liên quan.
    2. Step 2:
      1. Ở mục Rules, nhấn nút Add rules, chọn Add managed rules
        1. AWS managed rule groups:
          1. Paid: tạm thời bỏ qua
          2. Free: chọn Core rule set:
      2. Ở mục Default web ACL action for requests that don’t match any rules, chọn Allow
    3. Step 3: tạm thời do chỉ có 1 rule nên chúng ta sẽ chưa cần cấu hình priorities:
    4. Step 4: giữ nguyên các giá trị mặc định:
    5. Tiến hành khởi tạo sau khi đã review.

Sau khi khởi tạo công, chúng ta sẽ tiến hành đến phần chuẩn bị CloudFront & S3 cho static contents cũng như triển khai ứng dụng thông qua dịch vụ Elastic Beanstalk.
