Static Contents with CloudFront & S3

S3 block public access

  1. Truy cập vào dịch vụ S3.
  2. Ở thanh điều hướng, chọn Block Public Access settings for account:
  3. Nhấn nút Block all public access

S3 bucket for static contents

Chúng ta sẽ khởi tạo hai S3 buckets:

  • 1 cho việc chứa static contents cũng như sẽ là Origin của CloudFront distribution
  • 1 cho việc chứa logs của CloudFront distribution

Chúng ta sẽ lặp lại các bước sau để khởi tạo 2 buckets:

  1. Truy cập vào dịch vụ S3.
  2. Ở thanh điều hướng, chọn Buckets, nhấn nút Create bucket:
    1. Ở mục General configuration:
      1. Nhập Bucket name
      2. Chọn AWS Region tương ứng
    2. Ở mục Block Public Access settings, chọn Block all public access:
    3. Ở mục Default encryption:
      1. Encryption key type: Amazon S3-managed keys (SSE-S3)
      2. Bucket Key: Enable
    4. Tiến hành khởi tạo

Sau khi khởi tạo thành công, chúng ta sẽ có bucket như sau:

CloudFront

  1. Truy cập vào dịch vụ CloudFront.
  2. Ở thanh điều hướng, chọn Distributions, nhấn nút Create distribution.
    1. Ở mục Origin:
      1. Origin domain: chọn <s3-bucket-name>.s3.amazonaws.com đã khởi tạo
      2. Origin access: chọn Origin access control settings (recommended):
    2. Ở mục Default cache behavior:
      1. Compress objects automatically: Yes
      2. Viewer protocol policy: Redirect HTTP to HTTPS hoặc HTTPS only (phụ thuộc vào ứng dụng)
      3. Allowed HTTP methods: GET, HEAD
    3. Ở mục Cache key and origin requests:
    4. Ở mục Settings:
      1. Price class: Use North America, Europe, Asia, Middle East, and Africa
      2. AWS WAF web ACL: chọn WAF đã khởi tạo
      3. Alternate domain name (CNAME): thêm CNAME phụ thuộc vào ứng dụng
      4. Custom SSL certificate: chọn ACM certificate đã khởi tạo
      5. Standard logging: On và chọn S3 bucket đã khởi tạo

Sau cùng, chúng ta cần thêm vào S3 bucket policy nội dung tương tự sau:

{
   "Version": "2008-10-17",
   "Id": "PolicyForCloudFrontPrivateContent",
   "Statement": [
      {
            "Sid": "AllowCloudFrontServicePrincipal",
            "Effect": "Allow",
            "Principal": {
               "Service": "cloudfront.amazonaws.com"
            },
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::<s3-bucket-name>/*",
            "Condition": {
               "StringEquals": {
                  "AWS:SourceArn": "arn:aws:cloudfront::<aws-account-id>:distribution/<cloudfront-distribution-id>"
               }
            }
      }
   ]
}

Sau khi khởi tạo thành công, chúng ta có thể truy cập tới Distribution thông qua DNS mặc định để kiểm tra kết nối. Sau đó, chúng ta cần trỏ CNAME từ domain đến DNS này.
